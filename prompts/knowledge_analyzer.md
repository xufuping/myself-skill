# Knowledge & Value 分析 Prompt

## 任务

你将收到 **{name}** 的原材料（笔记、代码、文档、简历、口述等）。
从中提取知识体系与价值能力，用于构建 Knowledge & Value。

**原则：只提取有实际价值的内容。不要推断，有依据才写，没有就标注"原材料不足"。**

---

## 通用提取维度（所有人适用）

### 1. 专业技能图谱

- 核心技术栈/专业技能（精通 vs 熟悉 vs 了解）
- 工具链偏好（日常使用的工具、软件、平台）
- 专项能力（某个细分领域的深度）

```
输出格式：
核心技能：[列表 + 精通程度]
工具链：[列表]
专项能力：[描述]
```

### 2. 工作方法论

- 接到任务后的处理流程
- 做输出物（代码/文档/设计/分析）时的习惯和风格
- 质量标准（什么算"做完了"）
- 处理紧急问题的方式

```
输出格式：
接任务：[步骤]
输出习惯：[描述]
质量标准：[描述]
```

### 3. 知识体系

- 深度知识领域
- 常用的思考框架和方法论
- 踩坑经验（引用原话）
- 常引用/推荐的资源

```
输出格式：
领域：[列表]
方法论：[列表]
经验：["原话1", "原话2", ...]
推荐资源：[列表]
```

### 4. 人生经验

- 职业发展中的关键认知
- 学习方法和心得
- 生活技能和兴趣领域的深度知识

```
输出格式：
职业认知：[列表]
学习方法：[描述]
其他经验：[列表]
```

---

## 职业专项提取

根据 {name} 的职业，重点提取对应维度：

### 工程师（后端/前端/全栈/算法）

- 技术栈详情（语言、框架、中间件）
- 代码风格（命名习惯、注释风格、架构偏好）
- 接口/组件设计方式
- Code Review 关注点
- 部署运维习惯

### 产品经理/设计师

- PRD/设计文档结构偏好
- 优先级排序方法
- 用户研究方法
- 工具偏好

### 运营/市场

- 增长策略偏好
- 数据分析方法
- 内容创作风格
- 渠道经验

### 通用职业

如职业不在上述列表中，提取：
- 该领域的专业术语和核心技能
- 工作流程和输出标准
- 领域经验和判断框架

---

## 代码风格提取（如有 GitHub 仓库）

从代码仓库中提取：
- 技术栈使用频率
- 命名规范（变量/函数/文件）
- 注释习惯
- 架构偏好（模块化方式、设计模式）
- PR 描述风格
- Issue 讨论方式

---

## 输出要求

- 语言：中文
- 没有信息的维度：标注 `（原材料不足，建议追加相关文档/笔记）`
- 有原文依据的结论：加引号标注原话
- 输出结果直接用于生成 knowledge.md，要求具体可执行，不要写"可能""倾向于"
