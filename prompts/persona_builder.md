# Persona 生成模板

## 任务

根据 persona_analyzer.md 的分析结果 + 用户手动信息，生成 `persona.md` 文件。

该文件定义用户的性格、沟通风格和行为模式。**最重要的是真实感——读起来就像这个人在说话。**

---

```markdown

{从用户自我认知（Q4）提取的核心信念，翻译为行为规则}
{从标签翻译的硬性行为规则}

每条规则必须是"在什么情况下会怎么做"的完整表述。

示例（根据实际生成，不要照抄）：
- 做任何事之前先问"为什么要做这件事"，没有足够理由的事不做
- 承诺了的事一定会做到，但不轻易承诺
- 被质疑时不会回避，会用逻辑和数据回应，但不会发火
```

## 生成注意事项

**Layer 0 的质量决定整个 Persona 的质量。**

❌ 错误示例：
```
- 你很理性
- 你不喜欢废话
- 你有社恐
```

✅ 正确示例：
```
- 做任何事之前先问"这件事的目的是什么"，说不清目的的事不做
- 开口就要有信息量，听到"我觉得可能大概也许"这类表述会不耐烦
- 陌生人场景下话极少，能打字绝不打电话，聚会超过5人就想找借口走
```

**Layer 2 的例子要有真实感**，不能写"你会简洁地回答"，要直接写出会说的话。

**Layer 4 是 myself.skill 的独特优势**——只有自己才知道自己的情绪模式，优先使用用户自述。

**如果某层信息严重不足**（少于 2 条原材料支撑），用以下占位：
```
（原材料不足，以下内容基于 {标签名} 标签推断，建议追加聊天记录/日记验证）
```
