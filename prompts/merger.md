# 增量 Merge Prompt

## 任务

你将收到：
1. 现有的文件内容（knowledge.md / persona.md / identity.md）
2. 新的原材料内容

判断新内容应该更新哪个部分，输出增量更新内容。

**原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1：分类判断

将新内容中的每条信息归类：

| 信息类型 | 归入 |
|---------|------|
| 技术知识、技能、方法论、代码风格、项目经验 | → knowledge.md |
| 说话风格、口头禅、表达习惯 | → persona.md (Layer 2) |
| 思维方式、决策行为 | → persona.md (Layer 3) |
| 情绪表现、压力反应 | → persona.md (Layer 4) |
| 社交行为、人际关系 | → persona.md (Layer 5) |
| 基础信息变更（职业、所在地等） | → identity.md |
| 他人评价 | → social_mirror.md |
| 两者都有 | → 分别归入 |

---

## Step 2：检查冲突

对比新内容与现有内容：

- **补充**（新细节）→ 直接追加
- **确认**（已有信息）→ 忽略，不重复写
- **矛盾** → 输出冲突提示：

```
⚠️ 发现冲突：
- 现有：{现有描述}
- 新发现：{新内容描述}
- 来源：{文件名/时间}

建议：[保留现有 / 更新为新内容 / 两者都保留并标注时间]
请用户决定。
```

---

## Step 3：生成更新 Patch

对每个文件的更新：
```
=== {filename} 更新 ===

[追加到"{章节名}"节]
- {新内容}

[无更新] 或 [以上章节有更新]
```

---

## Step 4：生成更新摘要

```
本次更新摘要：
- knowledge.md：追加了 {N} 条新信息（{简要描述}）
- persona.md：追加了 {N} 条新信息（{简要描述}）
- identity.md：{有更新/无更新}
- 发现 {N} 处冲突，需要你确认（见上方）

版本将从 {vN} 升级到 {vN+1}。
确认应用更新？
```
