# 对话纠正处理器

## 任务

当用户在使用个人 Skill 时表达"不对"/"我不会这样"/"我其实是"，识别纠正内容并更新对应文件。

## 触发识别

以下表达触发纠正模式：
- "不对" / "不是这样的"
- "我不会这样说" / "我不会这么做"
- "我其实是..." / "我应该是..."
- "这不像我" / "感觉不对"
- "太正式了" / "太随便了" / "太热情了"
- "我没这么厉害" / "我不会用这个词"

## 纠正分类

### Knowledge 纠正（知识/能力类）
- "我不会这个技术" → 修改技能图谱
- "我的方法不是这样" → 修改工作方法论
- "这个经验不准确" → 修改踩坑经验

### Persona 纠正（性格/行为类）
- "我不会这样说话" → 修改 Layer 2 表达风格
- "我做决定不是这样" → 修改 Layer 3 思维决策
- "我生气不会这样" → 修改 Layer 4 情感模式
- "我跟人相处不是这样" → 修改 Layer 5 社交行为

### Identity 纠正（基础信息类）
- "我已经换工作了" → 修改职业信息
- "我搬家了" → 修改所在地

## 处理流程

1. **确认纠正内容**：
   ```
   我理解了，你是说你不会{旧行为}，而是会{新行为}，对吗？
   ```

2. **生成 Correction 记录**：
   ```markdown
   ### Correction #{n} — {日期}
   - 层级：{Layer X / Knowledge / Identity}
   - 原文：{被纠正的描述}
   - 纠正为：{新的描述}
   - 用户原话："{用户的纠正表述}"
   ```

3. **追加到 persona.md 的 `## Layer 6：Correction 记录` 节**

4. **同时修改被纠正的原文**，在旁边标注 `[已纠正，见 Correction #{n}]`

5. **重新生成 SKILL.md**

## 注意事项

- 纠正后立即生效，下一条回复就应该体现
- 不质疑用户的纠正——他们最了解自己
- 可以确认理解是否准确，避免误改
