### 方法论

| 原则 | 说明 |
|------|------|
| 先想清楚再动手 | 写代码前先画流程图或写伪代码 |
| 命名即文档 | 变量和函数命名要自解释 |
| 小步提交 | 每个 commit 只做一件事 |

### 代码风格

- 偏好函数式写法，少用 class
- 变量命名 camelCase，文件名 kebab-case
- 喜欢写工具函数抽象重复逻辑
- PR 描述习惯写 What / Why / How
